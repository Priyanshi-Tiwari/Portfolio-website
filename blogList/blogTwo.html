What is Semantic HTML?

HTML tags are classified into two types : Semantic elements and non-semantic elements.

Semantic means connected with meaning. So semantic HTML elements are those elements that describe their own meaning. Elements such as <header>,<footer>,<section> etc are considered semantic elements because they describe on their own what they are used for and the type of content that will be enclosed within these tags.

A semantic element describes its meaning to both the browser and the developer.

Non-semantic elements are the elements that tell you nothing about the content inside them.
Examples of non-semantic elements are <span> and <div>.


In older version of HTML people used the non-semantic <div> tag to describe elements. They used class or id to describe its purpose. For Example <div class=”header”>, <div class=”footer”>,<div class=”navigation”>.This looks okay when you have to write just a few lines of code but as a programmer, we have to write thousands of lines of code and this is when it becomes troublesome and difficult to read by both the browser and the developer. Here comes the semantic elements to your rescue.

    Examples of semantic elements:
  • <header>: It is the introductory section of the page
  • <footer>: It is located at the bottom of the page. It contains contact and copyright information, sitemaps etc.
  • <main>: It defines the main content of the document
  • <section>:It is used to define a section in the document. It can contain an introduction, content, contact information etc.
  • <nav>: It is used to define navigational links in the form of a navigation bar or menu.
  • <article>: It contains independent content which does not require any other context.
  • <aside>: It is used to place the content in the sidebar.
  • <figure>:It is used to add images,diagrams,illustrations etc.
  • <figcaption>: It is used to add a small caption.
  • <details>It adds additional information that user can view or hide.
  • <mark>: It is used to highlight the text.
  • <summary>: it defines a heading for the details element.

Advantages of using Semantic elements

1)It is easier to read.
For example: <h1> is a semantic element. It tells the browser that it is a top-level heading. The browser automatically increases the font of the text inside the h1 tag to make it look like a heading
<h1>This is a h1 heading</h1>
Or you can also write it like
<span style=“font-size:32px”>This is also a h1 heading</span>
But this does not have any semantic value. It does not tell anything about it’s meaning.

Using semantic elements not only makes it easier for the browser to read the code but the developer. It makes your code more consistent.

2)Improved search engine rankings: search engines take into account the semantic elements as important keyword and influence the rank of the page.

3)Makes it easier for the screen reader which helps the visually impaired people to navigate through the page. In conclusion, the semantic elements describe their own meaning and purpose and the type of content inside them. We should use it for better accessibility, readability and better indexing.
